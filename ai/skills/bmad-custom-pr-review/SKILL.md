# BMAD Custom PR Review (Stack-Aware)

Adversarial code review that understands stacked PRs. Automatically detects
whether the current branch is part of a Charcoal/gt stack, then reviews each
layer independently and posts findings directly to each PR.

---

## When to Use

- `review PR` / `review this PR` / `code review` â€” review the current branch's PR
- `review stack` / `review my stack` â€” review all PRs in the current stack
- `review open PRs` â€” review all open PRs authored by the current user

---

## Detection Phase (always run first)

### Step 1 â€” Detect forge (GitHub vs Azure DevOps)

```bash
REMOTE_URL=$(git remote get-url origin 2>/dev/null || true)
if echo "$REMOTE_URL" | grep -qiE "github\.com"; then
  FORGE="github"
else
  FORGE="azure"
fi
```

### Step 2 â€” Detect stack

```bash
# Check if Charcoal is initialized
GT_INITIALIZED=$(gt branch 2>/dev/null && echo "yes" || echo "no")

if [ "$GT_INITIALIZED" = "yes" ]; then
  # Get ordered stack layers (bottom-up: trunk first, current branch last)
  STACK_BRANCHES=$(gt log --oneline 2>/dev/null | awk '{print $1}' | head -20)
  STACK_DEPTH=$(echo "$STACK_BRANCHES" | wc -l | tr -d ' ')
else
  # Fall back to single-branch mode
  STACK_DEPTH=1
  STACK_BRANCHES=$(git branch --show-current)
fi
```

If `STACK_DEPTH > 1` â†’ **Stack Review Mode**
If `STACK_DEPTH == 1` â†’ **Single PR Review Mode**

## Stack Review Mode

### Step 3 â€” Build PR map

For each branch in the stack (skip trunk), resolve its PR number:

**Azure DevOps:**
```bash
az repos pr list \
  --source-branch <branch> \
  --organization "https://dev.azure.com/bofaz" \
  --project "Axos-Universal-Core" \
  --output json \
  | jq -r '.[0] | {id: .pullRequestId, title: .title, status: .status}'
```

**GitHub:**
```bash
gh pr list --head <branch> --json number,title,state | jq '.[0]'
```

### Step 4 â€” Parallel adversarial review (all layers concurrent)

Launch one `Agent(subagent_type="bmad-bmm-code-review")` per layer, all in
parallel. Each agent receives:

**Diff (incremental â€” only this layer's changes):**
```bash
git diff <parent_branch>..<this_branch>
```

**Context (background â€” parent layer summaries, NOT re-reviewed):**
```
Parent PRs in this stack (for context only â€” do not re-review these):
  PR #NNN (<branch>): <title>
  ...
```

**Additional stack-specific review instructions for each agent:**
```
Stack-layer checks (in addition to standard adversarial review):

1. INTERFACE CONTRACTS: Are the types, function signatures, and API contracts
   exposed by this layer consistent with what parent/child layers rely on?
   List any mismatches.

2. LAYER INDEPENDENCE: Can this PR be merged and built independently without
   its child PRs being merged first? Identify any dangling references.

3. INCREMENTAL TEST COVERAGE: Does this layer include tests for its own new
   code? Flag any coverage gaps that rely on fixtures or setup from parent
   layers that don't yet exist in isolation.

4. COMMIT ATOMICITY: Is the logical boundary between this layer and adjacent
   layers coherent? Flag if this layer's diff would be cleaner split or merged.

Apply standard adversarial review: find 3-10 specific, actionable problems.
NEVER respond with "looks good". Security, performance, and correctness first.
```

### Step 5 â€” Post findings to each PR

Collect each agent's review output. For each layer:

**Azure DevOps (create a review thread):**
```bash
az repos pr thread create \
  --id <PR_ID> \
  --comment "<REVIEW_FINDINGS>" \
  --organization "https://dev.azure.com/bofaz" \
  --project "Axos-Universal-Core"
```

**GitHub:**
```bash
gh pr review <PR_NUMBER> --comment --body "<REVIEW_FINDINGS>"
```

Format the comment with a clear header:
```markdown
## BMAD Adversarial Review â€” Layer N of M

**Branch:** `<branch>` â†’ `<parent_branch>`
**Scope:** Incremental diff only (N files, +X/-Y lines)

### Findings

[review findings here]

---
*Generated by bmad-custom-pr-review (stack-aware mode)*
```

## Single PR Review Mode

### Step A â€” Get current PR info

**Azure DevOps:**
```bash
CURRENT_BRANCH=$(git branch --show-current)
PR_INFO=$(az repos pr list \
  --source-branch "$CURRENT_BRANCH" \
  --organization "https://dev.azure.com/bofaz" \
  --project "Axos-Universal-Core" \
  --output json | jq '.[0]')
PR_ID=$(echo "$PR_INFO" | jq -r '.pullRequestId')
TARGET_BRANCH=$(echo "$PR_INFO" | jq -r '.targetRefName' | sed 's|refs/heads/||')
```

**GitHub:**
```bash
CURRENT_BRANCH=$(git branch --show-current)
PR_INFO=$(gh pr list --head "$CURRENT_BRANCH" --json number,baseRefName | jq '.[0]')
PR_NUMBER=$(echo "$PR_INFO" | jq -r '.number')
TARGET_BRANCH=$(echo "$PR_INFO" | jq -r '.baseRefName')
```

### Step B â€” Get diff and run review

```bash
DIFF=$(git diff "origin/$TARGET_BRANCH"...HEAD)
```

Launch `Agent(subagent_type="bmad-bmm-code-review")` with the diff and
standard adversarial review instructions (no stack-specific checks needed).

### Step C â€” Post findings

Post using the same ADO/GitHub commands as Step 5 above.

## Handling No Open PR

If no PR is found for a branch:
1. Print a warning: `No open PR found for branch <branch> â€” skipping comment post`
2. Still run and display the review locally
3. Continue with remaining layers
