# Goal NN — <short title>

## Objective

One concrete, observable outcome. Not a workstream, not a theme.

## Why

The motivation — what problem this solves, who asked for it, what it unblocks.

## Current state

What's true right now: what's already done, what's in progress, what's known and unknown.

## Non-goals

What this goal explicitly does NOT cover, to keep a fresh agent from over-scoping.

## Steps

Ordered, executable steps. A fresh agent with no other context should be able to follow these without guessing.

1. Step one.
2. Step two.

## Acceptance criteria

Observable, checkable conditions — specific files, reports, test results, counts, or explicit user sign-off. Not vibes.

## Evidence to update

Canonical paths this goal's work should update or produce (e.g. `output/<date>/report.md`, specific test suite, dashboard).

## Stop and ask if

Conditions under which a fresh agent should stop and get explicit user input rather than proceeding — e.g. before any production-touching action, on ambiguous acceptance criteria, on unresolved blockers.
